# Carpet CRM

AI-powered CRM для управления заказами мойки ковров. Оператор ведёт диалог
с клиентом в WhatsApp вручную, а в Telegram-боте пишет кодовое слово —
AI-агент (Anthropic API, Tool Use) считывает пересланный текст переписки
и сам создаёт заказ в базе данных. Дальше доставка и мойщики обновляют
статус заказа через тот же бот.

## Стек

- **FastAPI** — REST API и webhook для Telegram
- **aiogram 3** — Telegram-бот
- **PostgreSQL + SQLAlchemy 2.0 (async) + Alembic** — хранение данных и миграции
- **Anthropic API (Claude Haiku)** — извлечение данных заказа и команд из свободного текста
- **Docker Compose** — локальный запуск всего стека одной командой

## Архитектура

Проект построен слоями, чтобы бизнес-логика не зависела от того,
кто её вызывает — Telegram-бот, REST API или (в будущем) мобильное
приложение:

```
routers / handlers   -> входная точка: HTTP или Telegram-сообщение
        |
services             -> бизнес-логика ("создать заказ", "сменить статус")
        |
repositories          -> единственный слой, который трогает базу данных
        |
models                -> SQLAlchemy-модели
```

Если завтра появится мобильное приложение — оно вызывает тот же REST API,
который вызывает те же `services`, что и бот. Бизнес-логика не дублируется.

## Решения по архитектуре (и почему)

- **StrEnum для статусов заказа** — типобезопасность enum, но при этом
  сериализуется в JSON как обычная строка без доп. кода.
- **Статус хранится как `String` в БД, а не нативный PostgreSQL ENUM** —
  упрощает будущие миграции при добавлении новых статусов.
- **`carpet_count` в заказе nullable** — на момент создания заказа со слов
  клиента точное количество ковров может быть неизвестно; подтверждается
  доставкой при заборе.
- **`Client` — отдельная таблица**, а не поля внутри `Order` — избегаем
  дублирования данных клиента при повторных заказах (нормализация).
- **`server_default=func.now()`** вместо `default=datetime.now()` для
  timestamp-полей — время вставки определяет сама БД, а не Python-процесс.

## Установка и запуск (локально, без Docker)

```powershell
python -m venv venv
venv\Scripts\activate
pip install -e ".[dev]"
copy .env.example .env
# заполнить .env реальными значениями (токен бота, ключ Anthropic, БД)
```

## Структура проекта

```
src/
└──carpet_crm/
   ├── api/routes/     — HTTP-эндпоинты FastAPI
   ├── bot/handlers/   — обработчики сообщений Telegram
   ├── ai/             — AI-агент (Anthropic Tool Use)
   ├── core/           — конфигурация, безопасность
   ├── db/             — модели SQLAlchemy, enum'ы, подключение к БД
   ├── repositories/   — слой доступа к данным
   ├── schemas/        — Pydantic-схемы (валидация запросов/ответов)
   ├── services/        — бизнес-логика
   └── tasks/          — фоновые задачи (Celery, добавим позже)
alembic/            — миграции базы данных
tests/              — unit и integration тесты
```
